# Milestone 0 | Working in an Agile Team

## 0.9 Agile Ceremonies & Team Collaboration

### **_How does Kanban help manage priorities and avoid overload?_**

Kanban boards are visual project management tools, designed to map out items that ensure transparency throughout the development process. The board is commonly made up of four columns labelled "Backlog", "In Progress", "On Hold", and "Done":

- the "Backlog" column refers to the column of items that need to be started.
- the "In Progress" column refers to the column of tasks that are in the works
- the "On Hold" column contains items that cannot be moved due to external needs such as dependencies or reviews; and
- the "Done" column containing all finished tasks.

Since the backlog is and should be ordered, the topmost item of the list is of the highest priority, followed by the next highest priority task.

### **_How can you improve your workflow using Kanban principles?_**

Kanban visualizes the tasks to better understand their purpose in the whole project. Mapping these tasks makes the cognitive load visible. This also allows team members to focus on another task while waiting on tasks in the "On-Hold" column, ensuring to finish tasks and unblock existing work. Additionally, it also allows the workflow to be manageable, prioritizing pending tasks on a column for it to finish the overall workflow. Finally, these tasks must also be explicitly defined, such that tasks don't revert back to the flow of the Kanban architecture.

Github has a built-in project feature that generates a KanBan board: [Github Kanban Board](assets/github-kanban.png). Github also has automated workflows enable that automates open issues and pull requests into the to-do column, and other automated workflows. Adding more of these workflows would greatly improve task tracking as the project soon evolves.
