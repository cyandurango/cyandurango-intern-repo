# Milestone 2 | Set up tools

## 2.5 Set up VS Code and learn keyboard shortcuts

Goal: Research VS Code shortcuts and list at least 10 shortcuts that you find useful. Categorize them into Navigation, Editing, Debugging, and Terminal Commands.

### **_My Top 10 Keyboard Shortcuts List_**

For Navigation:

- [`CMD + F`](assets/shortcut-results-cmd-f.png): useful for finding certain variables and keywords in a large codebase
- [`CTRL + G`](assets/shortcut-results-ctrl-g.png): 'go to line' command
- [`CMD + OPT + arrow up/down`](assets/shortcut-results-cmd-opt-up-down.mov) cursor selection inserted above or below selected line

For Editing:

- [`SHIFT + OPT + <-/->`](assets/shortcut-results-shift-opt-left-right.png) useful for selecting certain words and lines of code carefully.
- [`OPT + SHIFT + A`](assets/shortcut-results-opt-shift-a.mov) generates comment blocks for documentation purposes
- [`OPT + arrow up/down`](assets/shortcut-results-opt-up-down.mov) move whole line of code up or down the codebase

For Debugging

- `F5` start/continue the program
- `SHIFT+F5` stop the program

For Terminal Commands:

- [`CMD + arrow up/down`](assets/shortcut-results-cmd-up-down.mov) useful in scrolling through terminal output lines.
- [``CTRL + ` ``](assets/shortcut-results-ctrl-apostrophe.mov) show/hide integrated terminal

### **_Which shortcuts do you think will be the most useful in your daily work?_**

The editing and navigation shortcuts would be the most useful, as it would certainly help in viewing and changing the code much easier. The most useful command out of all commands, for me, would be the selection command (`SHIFT + OPT + <-/->`). Selection via mouse/trackpad takes too much time to transfer from keyboard to mouse than back to keyboard, and it usually overshoots.

### **_How can keyboard shortcuts improve your productivity?_**

Keyboard shortcuts takes less time as it keeps your hands in the keyboard at all times, allowing no time lags from moving your hands toward the mouse or touchpad.

Many times I have used `CMD + F` in finding certain variables or function names in a function, especially in a very long codebase. Pressing `RETURN` or `SHIFT + RETURN` would let me skim through all words inside the text box. It was handy in scouring through the code for that specific function.
